# Database Schema Design: Review Analysis System

This document outlines the detailed relational database schemas (or collection fields) used to store raw feedback, processing workflows, classifications, summaries, and user interview data.

---

### 1. raw_reviews
Stores the raw, unprocessed user feedback directly as retrieved from external sources.

| Field | Data Type | Purpose | Example Value |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary key; uniquely identifies each raw feedback entry. | `8f2a1b9e-4c7b-402a-9e12-3f8b654012ab` |
| `source` | VARCHAR | The source platform of the review. | `'reddit'` or `'app_store'` |
| `raw_text` | TEXT | The raw, unformatted text of the review. | `"I hate Spotify's search page. It's so cluttered with podcasts! <br> Fix it."` |
| `rating` | INTEGER (Nullable) | App Store review rating score, or null for forum posts. | `2` |
| `author` | VARCHAR | Username or user ID on the originating platform. | `"music_lover_99"` |
| `retrieved_at` | TIMESTAMP | Timestamp of when the review was pulled into the system. | `2026-06-25 23:53:36.123` |
| `metadata` | JSON | Key-value store for source-specific parameters. | `{"app_version": "8.8.96", "subreddit": "spotify", "thread_id": "18ajfh"}` |

---

### 2. cleaned_reviews
Contains normalized plain-text review data after removing noise and formatting artifacts.

| Field | Data Type | Purpose | Example Value |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary key. | `cd0e4f21-72f5-46aa-bb94-11ccdd88ee99` |
| `raw_review_id` | UUID | Foreign key referencing `raw_reviews.id`. | `8f2a1b9e-4c7b-402a-9e12-3f8b654012ab` |
| `cleaned_text` | TEXT | Stripped, normalized plain-text content. | `"I hate Spotify's search page. It's so cluttered with podcasts! Fix it."` |
| `detected_language`| VARCHAR | ISO 639-1 language code of the review. | `"en"` |
| `is_noise` | BOOLEAN | Flag indicating whether the review is empty or spam. | `false` |
| `cleaned_at` | TIMESTAMP | Timestamp of when the cleaning step ran. | `2026-06-25 23:54:10.450` |

---

### 3. classified_reviews
Stores the results of AI categorization, sentiment scoring, and user segment modeling.

| Field | Data Type | Purpose | Example Value |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary key. | `ff88aa22-bb11-44ee-aa22-001122334455` |
| `cleaned_review_id`| UUID | Foreign key referencing `cleaned_reviews.id`. | `cd0e4f21-72f5-46aa-bb94-11ccdd88ee99` |
| `primary_label` | VARCHAR | Assigned primary feedback tag. | `"discovery frustration"` |
| `sublabels` | VARCHAR[] (Array) | Sub-topics parsed from the text. | `["ui-clutter", "search-friction"]` |
| `sentiment_score` | FLOAT | Polarity score between -1.0 (very negative) and +1.0 (very positive). | `-0.75` |
| `inferred_segment` | VARCHAR | Modeled user category based on context/metadata. | `"Premium/iOS Power User"` |
| `confidence` | FLOAT | Classification confidence level between 0.0 and 1.0. | `0.94` |
| `workflow_run_id` | UUID | Foreign key referencing `workflow_runs.id`. | `bb22cc44-dd66-44ee-99aa-887766554433` |
| `classified_at` | TIMESTAMP | Timestamp of when classification was finalized. | `2026-06-25 23:54:15.980` |

---

### 4. summary_insights
Stores periodic, aggregated reports generated by the AI for product managers.

| Field | Data Type | Purpose | Example Value |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary key. | `dd55ee11-22aa-33bb-44cc-55ddeeff0011` |
| `time_period_start`| TIMESTAMP | The starting timestamp of the aggregation window. | `2026-06-18 00:00:00.000` |
| `time_period_end` | TIMESTAMP | The ending timestamp of the aggregation window. | `2026-06-25 23:59:59.999` |
| `source_channel` | VARCHAR | Channel filter applied (or `'all'`). | `'all'` |
| `avg_sentiment` | FLOAT | Mean sentiment score over the specified period. | `-0.15` |
| `top_themes` | JSONB (Array) | Top labels and their respective occurrences. | `[{"theme": "discovery frustration", "count": 248}, {"theme": "missing control", "count": 120}]` |
| `summary_text` | TEXT | Narrative weekly AI-written report summary. | `"Users are reporting a high volume of frustration with the Home tab podcast recommendation clutter..."` |
| `action_items` | TEXT[] (Array) | Bullet points of recommended design or engineering fixes. | `["Add a 'hide podcast' context menu", "Optimize search indexing for local artists"]` |
| `created_at` | TIMESTAMP | Timestamp of when the report was compiled. | `2026-06-25 23:55:00.000` |

---

### 5. workflow_runs
Tracks pipeline executions for audit, tracking, and debugging.

| Field | Data Type | Purpose | Example Value |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary key. | `bb22cc44-dd66-44ee-99aa-887766554433` |
| `started_at` | TIMESTAMP | Start time of the execution. | `2026-06-25 23:54:05.000` |
| `completed_at` | TIMESTAMP (Nullable) | Completion time of the execution. | `2026-06-25 23:54:20.000` |
| `status` | VARCHAR | Status of execution (`'running'`, `'completed'`, `'failed'`). | `"completed"` |
| `processed_count` | INTEGER | Number of feedback records analyzed. | `150` |
| `error_message` | TEXT (Nullable) | Error output if run failed. | `null` |

---

### 6. user_interviews
Links qualitative user research sessions to quantitative review data.

| Field | Data Type | Purpose | Example Value |
| :--- | :--- | :--- | :--- |
| `id` | UUID | Primary key. | `aa99ee11-22ff-33bb-44dd-55ccaa668899` |
| `user_id` | VARCHAR | Target user identification code. | `"spotify_user_9214"` |
| `interview_date` | TIMESTAMP | Date the user research session took place. | `2026-06-25 14:00:00.000` |
| `transcript` | TEXT | Raw transcript lines from the interview. | `"Q: Why did you rate the search screen 2 stars? A: Too much junk on the home screen..."` |
| `summary_notes` | TEXT | Bullet points highlighting the user's core feedback. | `"- User feels overwhelmed by podcast recommendations.<br>- Expresses need to block artists."` |
| `linked_reviews` | UUID[] (Array) | Foreign keys pointing back to reviews that triggered the interview. | `["8f2a1b9e-4c7b-402a-9e12-3f8b654012ab"]` |
| `created_at` | TIMESTAMP | Creation timestamp of the record. | `2026-06-25 23:56:00.000` |
