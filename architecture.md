# System Architecture: Spotify Growth AI Review Discovery Engine

This document details the system architecture of the AI-powered user feedback discovery engine.

| Component | Job | Input Data | Output Data | Connections |
| :--- | :--- | :--- | :--- | :--- |
| **Frontend** | Serve as the interactive dashboard for Spotify Product Managers to search, filter, and view user feedback trends and AI summaries. | User interactions (search inputs, filters, clicks); JSON responses from Backend APIs. | HTTP REST requests; rendered HTML/CSS/JS UI elements; visual analytics charts. | Connects to the **Backend** API via HTTPS. Deployed on the **Deployment Target** (static hosting/CDN). |
| **Backend** | Act as the API gateway, orchestrate tasks, run scheduled ingestions from App Store/Play Store/Reddit/forums, and query the Database. | HTTP requests from Frontend; raw scraping/API data from source channels; database queries. | JSON payloads (feedback lists, trends, status info); raw data queues for the workflow engine; DB read/write statements. | Handles requests from the **Frontend**; schedules jobs via the **Workflow Engine**; reads/writes to the **Database**. |
| **Workflow Engine** | Orchestrate ingestion pipelines, handle failure retries, chain prompts, and coordinate transformations between raw text and structured analysis. | Raw feedback items from Backend (payloads containing text, channel, timestamp). | Structured, AI-analyzed JSON records (sentiment, tags, categories, summarized insights). | Triggered by the **Backend**; queries/calls the **AI Model Layer**; saves processed results to the **Database**. |
| **Database** | Provide persistent, indexed storage for raw feedback, structured tags, and aggregated sentiment histories. | Write/Update queries from Backend or Workflow Engine containing raw and enriched feedback. | Row sets or JSON records matching query criteria requested by the Backend. | Accessible by the **Backend** (for serving reads) and the **Workflow Engine** (for updating enriched results). |
| **AI Model Layer** | Perform Natural Language Processing (NLP) to classify text sentiment, extract topics, and generate summaries. | Prompts containing raw text, category definitions, and completion format instructions. | Natural language responses containing sentiment, classification tags, and summary strings. | Called via secure HTTPS API requests (e.g., OpenAI/Anthropic APIs) by the **Workflow Engine**. |
| **Deployment Target** | Provide the runtime environment, compute, database hosting, static file serving, and networking infrastructure. | Code build packages; environment variables; Docker configuration/images. | Live web addresses (URLs); resource metrics; service execution logs. | Hosts and networks the **Frontend**, **Backend**, **Workflow Engine**, and **Database** together. |
