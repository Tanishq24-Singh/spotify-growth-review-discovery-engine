# Frontend Screen Designs: Spotify Growth Discovery Engine

This document details the interface layouts, functional behaviors, input/output data, and user actions for the frontend dashboard screens.

---

### 1. Upload or Trigger Workflow Screen
*   **Purpose:** Allow Product Managers to manually import raw user reviews (via bulk upload) or trigger live API ingestion cycles.
*   **Main UI Elements:**
    *   *Source Selector Checklist:* Checkboxes for Google Play Store, iOS App Store, Reddit, and Web forums.
    *   *Date Range Picker:* Calendar widget to select the start and end ingestion boundaries.
    *   *Drag-and-Drop Ingestion Zone:* File uploader for custom CSV or JSON review lists.
    *   *Primary Action Button:* Prominent, glowing "Run AI Discovery Pipeline" button.
*   **Data it Shows:**
    *   Current database feedback volume stats (e.g., "12,450 Reviews stored").
    *   Timestamp and source of the last successful data collection run.
*   **User Actions:**
    *   Configure data sources and dates for ingestion.
    *   Upload local CSV/JSON review datasets.
    *   Trigger the backend ingestion and classification workflow.

---

### 2. Workflow Status Screen
*   **Purpose:** Monitor pipeline progress in real-time, inspect processing throughput, and debug runtime failures.
*   **Main UI Elements:**
    *   *Active Step Timeline:* Horizontal node stepper displaying active status (e.g., Ingesting -> Cleaning -> Classifying -> Summarizing).
    *   *Progress Bar & Counter:* Shows percentage completion and numeric metrics (e.g., "Classified 87 of 150 reviews").
    *   *Console Log Terminal:* Collapsible black box outputting live API execution lines.
    *   *Emergency Stop Trigger:* Button to cancel active workflow runs.
*   **Data it Shows:**
    *   Active job state (Pending, Processing, Completed, or Failed).
    *   Current error stack traces and warnings if a step fails.
    *   Elapsed execution runtime.
*   **User Actions:**
    *   Cancel/kill a running pipeline instance.
    *   Rerun/retry failed extraction jobs.
    *   Export execution logs for system diagnostics.

---

### 3. Insights Dashboard Screen
*   **Purpose:** Provide PMs with a high-level analytics cockpit to monitor weekly sentiment shifts, top discovery complaints, and AI-suggested product changes.
*   **Main UI Elements:**
    *   *KPI Card Grid:* Summary counts (Net Sentiment Score, Total Analyzed, Segment Pain Ratios).
    *   *Sentiment Trend Chart:* Interactive line chart tracking weekly sentiment scores over time.
    *   *Theme Distribution Graph:* Bar chart displaying primary themes (e.g., discovery frustration, repetitive listening).
    *   *Action Plan Panel:* Card layout highlighting AI-recommended design and engineering tasks with Priority levels.
*   **Data it Shows:**
    *   Aggregated weekly/monthly averages of sentiment and topic tags.
    *   Automated text executive summaries generated by the AI compiler.
*   **User Actions:**
    *   Filter all dashboard graphs by date ranges, source channel, or user segment.
    *   Click on a theme chart bar to deep-dive into that specific review segment.
    *   Upvote/downvote recommended product action items.

---

### 4. Review Themes Table Screen
*   **Purpose:** Provide a tabular audit list of all ingested reviews, allowing managers to sort, filter, and drill down into individual customer testimonials.
*   **Main UI Elements:**
    *   *Semantic Search Input:* Text box with auto-suggestions.
    *   *Filter Ribbon:* Multi-select filters for primary label, sublabels, sentiment tier (Positive/Neutral/Negative), and user segment.
    *   *Interactive Data Grid:* Column-sortable table (columns: Date, Author, Source, Rating, Primary Label, Sentiment, Segment).
*   **Data it Shows:**
    *   Paginated rows of analyzed feedback records including dates, text snippets, assigned labels, and confidence levels.
*   **User Actions:**
    *   Input queries to search text records.
    *   Filter and sort reviews by rating, sentiment score, or label categories.
    *   Click any review row to open the sidebar detail drawer.

---

### 5. Example Quotes Panel (Detail/Sidebar Drawer)
*   **Purpose:** Display the full text of a selected review, highlighting the specific evidence quotes and metadata used by the AI to make classification decisions.
*   **Main UI Elements:**
    *   *Sliding Drawer Canvas:* Panel that opens on the right side of the screen.
    *   *Highlighted Text Viewer:* Raw review body text with the target `evidence_quote` highlighted in yellow.
    *   *Metadata Badge Row:* Pills indicating source, rating, and classification confidence.
    *   *AI Re-evaluation Form:* Dropdowns for manually overriding incorrect labels.
*   **Data it Shows:**
    *   Complete review text, user metadata parameters (app version, device model).
    *   Assigned classifications, sentiment scores, and confidence percentages.
*   **User Actions:**
    *   Override incorrect AI labels to retrain or adjust classification logs (human-in-the-loop).
    *   Copy the highlighted evidence quote to clipboard.
    *   Tag this review to trigger an offline user research interview request.

---

### 6. Summary Export Panel
*   **Purpose:** Package synthesized AI insights and recommendations into shareable, print-ready reports.
*   **Main UI Elements:**
    *   *Interactive Document Previewer:* Standardized document layout showcasing executive summaries, tables, and charts.
    *   *Export Control Bar:* Clickable buttons for "Download PDF", "Export Markdown", and "Copy JSON Payload".
    *   *Distribution Form:* Input fields to email or webhook the report directly to Slack/Teams channels.
*   **Data it Shows:**
    *   Preview representation of the final aggregated insights report (barriers, goals, unmet needs, action points).
*   **User Actions:**
    *   Toggle layout section checkboxes (e.g., exclude/include raw quote lists in export).
    *   Initiate downloads of PDF, Markdown, or JSON formats.
    *   Send the report to other product team members.
